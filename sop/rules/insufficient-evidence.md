# 근거 부족 시 처리 절차 (Insufficient Evidence Protocol)

## 목적

"근거 불충분" 판정의 신뢰도를 확보하기 위해, 검색 중단 기준과 보고 양식을 정의한다. "찾지 못했다"와 "존재하지 않는다"를 명확히 구분한다.

---

## 1. 최소 검색 요건 (Minimum Search Requirements)

"근거 불충분"으로 판정하기 전에, 항목별로 아래 최소 검색을 완료해야 한다.

### 필수 조사 항목 (항목 1~5)

| 요건 | 기준 |
|------|------|
| **최소 검색 DB 수** | 3개 이상의 독립 데이터베이스 |
| **최소 검색 쿼리 수** | 5회 이상 (별칭 포함 조합) |
| **별칭 사용** | 모든 공식 별칭을 최소 1회 이상 검색에 포함 |
| **출판 연도 제한 해제** | 연도 필터 없이 전체 기간 검색 1회 이상 |

### 선택 조사 항목 (GWAS, GTEx 등)

| 요건 | 기준 |
|------|------|
| **최소 검색 DB 수** | 2개 이상 |
| **최소 검색 쿼리 수** | 3회 이상 |

---

## 2. 검색 중단 판정 기준

아래 조건 중 **하나**를 만족하면 해당 항목의 검색을 중단하고 "근거 불충분"으로 판정할 수 있다.

| 조건 | 설명 |
|------|------|
| **완전 미검출** | 최소 검색 요건을 충족한 후에도 관련 결과가 0건 |
| **비특이적 결과만 존재** | 검색 결과가 대상 유전자를 직접 다루지 않고, 일반적 경로/질병 정보만 반환 |
| **종 불일치** | 결과가 존재하나 조사 대상 종이 아닌 다른 종의 데이터만 존재 |
| **조직 불일치** | 결과가 존재하나 조사 대상 조직/기관이 아닌 다른 맥락의 데이터만 존재 |

---

## 3. "근거 불충분" 보고 양식

판정 시 반드시 아래 양식으로 기재한다.

```markdown
### [항목명]

> **근거 수준**: [Insufficient]
>
> **검색 범위**:
> - 검색 DB: PubMed, GWAS Catalog, IMPC, GTEx
> - 검색 쿼리 수: 7회
> - 사용 키워드: "LIN7C", "MALS-3", "VELI3" × "obesity", "adipose", "BMI"
> - 검색 기간: 전체 기간 (연도 제한 없음)
>
> **판정 사유**: 상기 검색 범위에서 대상 유전자와 비만 간의 직접적 연관을 보고한
> 연구가 검출되지 않음
>
> **주의**: 본 판정은 검색 시점(YYYY-MM-DD) 기준이며,
> 향후 신규 연구 발표에 의해 변경될 수 있음
```

## 4. "근거 불충분"과 "음성 결과"의 구분

| 구분 | 의미 | 보고 표현 |
|------|------|----------|
| **근거 불충분 (Insufficient)** | 관련 연구 자체가 수행되지 않았거나 검색되지 않음 | "관련 연구가 확인되지 않음" |
| **음성 결과 (Negative)** | 연구가 수행되었으나 유의미한 연관/효과가 관찰되지 않음 | "X 연구에서 유의미한 연관이 관찰되지 않음 (출처)" |

- **음성 결과**는 그 자체로 유의미한 정보이므로, 출처와 함께 상세히 기술한다.
- **근거 불충분**은 부재의 증거가 아니라 증거의 부재이므로, 단정적 표현을 피한다.

---

## 5. 에스컬레이션 (Escalation)

아래 경우, Researcher 에이전트는 독자적으로 "근거 불충분" 판정을 내리지 않고 Manager에게 에스컬레이션한다.

| 상황 | 조치 |
|------|------|
| 간접 증거는 다수 존재하나 직접 증거가 없는 경우 | Manager가 간접 증거의 종합 해석 여부를 판단 |
| 인접 유전자의 데이터가 풍부하여 대상 유전자 기여도 판단이 필요한 경우 | Manager가 유전자좌 분리 분석 필요성을 판단 |
| 프리프린트(preprint)만 존재하고 peer-review 논문이 없는 경우 | Manager가 프리프린트 인용 여부를 판단 |
